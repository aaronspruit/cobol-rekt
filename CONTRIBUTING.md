# Contributing to Cobol-REKT

Thank you for your interest in contributing to Cobol-REKT! We welcome contributions from the community and appreciate your help in making this project better. Below are some guidelines to help you get started.

# Contributing

When contributing to this repository, please first discuss the change you wish to make via issue, email, or any other method with the owners of this repository before making a change.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. Increase the version numbers in any examples files and the README.md to the new version that this Pull Request would represent. The versioning scheme we use is [SemVer](http://semver.org/).
4. Your Pull Request will be merged once you have the sign-off of the developer(s).
